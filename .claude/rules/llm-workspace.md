# LLM Workspace

`CLAUDE.md` (project root) is your operational constitution — read it at session start. It describes the full workflow, task lifecycle, and `llm/` directory layout.

## Quick Reference

- `.devtool/features/` — Kanban board for task tracking (managed via `/kanban-markdown` skill)
- `llm/ecosystem/` — Stable cross-cutting reference (overview, projects, data-pipeline, deps, deployment, upstream-updates)
- `llm/projects/<name>/` — Deep dives for specific software projects
- `llm/scratch/` — Throwaway scripts

## Maintenance

- When something is wrong, fix it. A wrong doc is worse than no doc.
- When structure changes, update `CLAUDE.md` first.
